All Questions

There are a number of common attributes to be set for all questions.

Label
Usually, the number to the left of the question. For example, Title and Instruction usually don't have a label (though they may).
Sort Order
An the number by which questions will be ordered in the eval.
To get a list of the questions, their IDs, sort orders, and a summary, choose the "quick reference" link at the top of the question editing page.
Required
A checkbox to indicate whether the question should be required. Required questions will have a red box next to them in the user form when they haven't been answered, and users will not be allowed to submit evals without answering all required questions.
Not for Title, Instruction.
N/A Available
A checkbox to indicate whether or not "N/A" should be given as a response choice.
Not for Title, Instruction, MultipleResponse, FillIn.
Grouping
Inidcates which other question to cross-reference the results of this question against. Only certain discrete-answer questions will show up as choices.
To get a list of the questions, their IDs, sort orders, and a summary, choose the "quick reference" link at the top of the question editing page.
(Not for Title, Instruction.)
Text
The text of the question (or title or instruction) itself. This is required to be well-formed XML (that is, all tags must be closed). Text is not required to be in <para> tags, though it may be. On saving to the database, HTML tags are converted to HSCML equivalents. The following HTML tags (and their HSCML equivalents) are allowed in eval question text bodies.
HTML HSCML
<b> <strong>
<i> <emph>
<br> <break>
<table> <table>
<tr> <tr>
<th> <th>
<td> <td>
<ul> <itemized-list>
<ol> <enumerated-list>
<li> <list-item>

Multiple Choice, Multiple Response, Discrete Numeric

Columns
The number of columns the choices should display in.
Choices
The text of the choices themselves. Empty choice boxes will be ignored.

Numeric Rating, Plus/Minus Rating

Num Steps
How many steps the rating range should be. Should be odd. If this value is 5, Numeric Rating will be interpreted as 1 to 5, and Plus/Minus Rating would be interpreted as -2 to +2.
Show Numbers
Whether or not to show numbers next to the choices in the question.
Style
Whether to use radio buttons ("Radiobox") or a drop-down box ("Dropdown").
Low Text
What the label should be a the low (leftmost) end of the scale.
Mid Text
What the label should be on the middle of the scale.
High Test
What the label should be on the high (rightmost) end of the scale.

Count

Low Bound
What the bottom of the lowest interval given as a choice should be.
Interval
How broad the intervals should be.
High Bound
What the top of the highest interval should be.
And Lower?
A check box indicating whether a choice should be given for quantities lower than the low bound.
And Higher?
A check box indicating whether a choice should be given for quantities higher than the high bound.

For example, to get the choices "0 to 2", "3 to 5", "6 to 8" , and "More than 8", one would choose a low bound of 0, a high bound of 8, an interval of 3. "And lower?" would be unchecked, and "And Higher?" would be checked.

Fill-In

Long text
A checkbox to indicate whether a long answer is expected. When the box is checked, users will get a word-wrapped, multi-line text box.

Question Reference

Target Question ID
The ID of the other question to which this question is a reference. Only questions which are "targetable" are listed in the drop down box.
